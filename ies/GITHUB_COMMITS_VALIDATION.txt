CLUSTER CHARACTERISTICS
─────────────────────────────────────────────────────────────────────────────────

BRIEF CLUSTER (15-25 chars):
  Examples:
    • "Fix typo"
    • "Update docs"
    • "Minor fix"
  Function: Quick bug fixes, documentation updates
  Volume: ~46-47% of commits (high)
  Synergy: Efficient (dense information)

MODERATE CLUSTER (40-44 chars):
  Examples:
    • "Add feature for user authentication"
    • "Improve performance of list sorting"
    • "Refactor database connection logic"
  Function: Feature additions, refactoring
  Volume: ~37-39% of commits (high)
  Synergy: Balanced (detail + brevity)

DETAILED CLUSTER (64-71 chars):
  Examples:
    • "Merge pull request #123: Add support for async operations"
    • "Update changelog with version 2.0 release notes"
    • "Fix critical bug that prevented database connections"
  Function: Merges, major changes, documentation
  Volume: ~17-23% of commits (moderate)
  Synergy: Comprehensive (full context)

INTERPRETATION:
  Three commit styles have DISTINCT functions in the repository
  Just like three communication roles in IES community
