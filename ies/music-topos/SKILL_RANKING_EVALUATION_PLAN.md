# Skill Quality Ranking: Evaluation Framework

## Evaluation Criteria (Multi-Dimensional)

### A. Introspective Context Score (Music-Topos Specific)
**Weight: 35%**
- Relevance to categorical music composition (0-10)
- Integration with reafference/corollary discharge system (0-10)
- Support for CRDT/temporal versioning (0-10)
- Color dynamics and deterministic generation (0-10)
- Glass Bead Game integration potential (0-10)

### B. Generic Quality Measures (Exa Research)
**Weight: 50%**
- Community adoption/GitHub stars (0-10)
- Maintenance status/recency (0-10)
- Documentation quality (0-10)
- Security track record (0-10)
- Developer experience/ease of use (0-10)

### C. Ecosystem Fit
**Weight: 15%**
- Synergy with other skills (0-10)
- Cross-agent collaboration potential (0-10)

## Research Methodology

### Per-Skill Process:
1. **Read SKILL.md** to understand context and purpose
2. **Exa Search** with 77 results (7 passes × 11 results) on:
   - Skill name + "GitHub stars quality"
   - Skill name + "documentation"
   - Skill name + "security vulnerabilities"
   - Skill name + "adoption usage"
   - Skill name + "maintenance"
   - Skill name + "community"
   - Skill name + "best practices"
3. **Aggregate scores** from research
4. **Contextual evaluation** specific to music-topos

### Scoring Scale:
- 0-2: Poor/Minimal
- 3-4: Below Average
- 5-6: Average
- 7-8: Good/Strong
- 9-10: Excellent/Critical
