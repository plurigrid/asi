## I. Core Principle: Triadic Balance

### GF(3) Conservation Rule

```
For any valid triad (S₊, S₋, S₀):
  trit(S₊) + trit(S₋) + trit(S₀) = 1 + (-1) + 0 = 0 ✓
```

### Semantic Constraints

Beyond pure arithmetic, valid triads must satisfy:

1. **Functional Coherence**: S₊ generates, S₋ validates, S₀ coordinates
2. **Dependency Respect**: No circular dependencies between triad members
3. **Scope Overlap**: Skills should operate on compatible data/domains
4. **Composition Feasibility**: Output of one feeds input of next

## IV. Systematic Generation Algorithm

### Algorithm: Generate Valid Triads

```
Input:
  PLUS_SKILLS = {S₁, S₂, ..., S₄₄}     [44 generative skills]
  MINUS_SKILLS = {T₁, T₂, ..., T₇₁}    [71 validating skills]
  ERGODIC_SKILLS = {U₁, U₂, ..., U₈₅}  [85 coordinating skills]

Output:
  VALID_TRIADS = {(Sᵢ, Tⱼ, Uₖ) | constraints satisfied}

Algorithm:
  for each Sᵢ in PLUS_SKILLS:
    for each Tⱼ in MINUS_SKILLS:
      for each Uₖ in ERGODIC_SKILLS:
        if FUNCTIONAL_COHERENCE(Sᵢ, Tⱼ, Uₖ) and
           DEPENDENCY_RESPECT(Sᵢ, Tⱼ, Uₖ) and
           SCOPE_OVERLAP(Sᵢ, Tⱼ, Uₖ) and
           COMPOSITION_FEASIBLE(Sᵢ, Tⱼ, Uₖ):
          VALID_TRIADS.add((Sᵢ, Tⱼ, Uₖ))

Result:
  |VALID_TRIADS| ≈ 500-800 semantically valid compositions
```

### Constraint Functions

```
FUNCTIONAL_COHERENCE(S₊, S₋, S₀):
  return (S₊.outputs compatible with S₀.inputs) AND
         (S₀.outputs compatible with S₋.inputs) AND
         (S₋.outputs provide feedback to S₊)

DEPENDENCY_RESPECT(S₊, S₋, S₀):
  return NOT(S₊ depends on S₋ or S₋ depends on S₊) AND
         (No circular dependencies in composition)

SCOPE_OVERLAP(S₊, S₋, S₀):
  return (S₊, S₋, S₀ operate on compatible domains) AND
         (Share common types/ontologies)

COMPOSITION_FEASIBLE(S₊, S₋, S₀):
  return (Can chain operations) AND
         (No data loss in transformation)
```
